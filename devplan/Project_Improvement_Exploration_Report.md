#### [P1-1] 핵심 명령 레이어 테스트 확장
| 항목 | 내용 |
|------|------|
| **ID** | `test-commands-001` |
| **카테고리** | 🧪 테스트 |
| **복잡도** | Medium |
| **대상 파일** | `src/components`, `src/hooks`, `src/services`, `src/__tests__` |
| **Origin** | static-analysis / manual-idea |
| **리스크 레벨** | critical |
| **관련 평가 카테고리** | testCoverage, codeQuality |

**현재 상태:** 테스트 스크립트만 있고 실제 테스트 파일이 거의 없으며, 특히 견적 계산 및 리비전 저장과 같은 핵심 로직이 보호받지 못하고 있음.

**문제점 (Problem):** 작은 코드 수정에도 주요 기능이 오작동할 위험(회귀)이 크며, 이를 검증할 자동화 수단이 없음.

**영향 (Impact):** 배포 후 치명적 버그 발생 가능성, 유지보수 비용 급증.

**원인 (Cause):** 초기 개발 시 기능 구현 속도에 치중하여 테스트 작성을 생략함.

**개선 내용 (Proposed Solution):** Jest와 React Testing Library를 사용하여 `useProjectDetail` 훅과 `ProjectFormModal` 컴포넌트의 통합 테스트를 작성하고, `npm test`로 검증 가능한 환경 구축.

**기대 효과:** 리팩토링 안정성 확보, 버그 조기 발견.

**Definition of Done:**
- [ ] `src/__tests__` 폴더 생성 및 기본 테스트 설정
- [ ] 핵심 비즈니스 로직(견적 계산) 단위 테스트 구현
- [ ] 주요 사용자 흐름(프로젝트 생성/수정) 통합 테스트 구현
- [ ] CI/로컬에서 테스트 통과 확인
